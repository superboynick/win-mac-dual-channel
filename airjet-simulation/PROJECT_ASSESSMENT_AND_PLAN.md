# AirJet 仿真论文：项目评估与执行计划

评估日期：2026-07-12  
对应主文件：`AIRJET_SIMULATION_PROJECT.md`  
结论：**可以做，且应立即开始；但首篇论文必须收缩为“经验证的 AirJet 类脉冲微射流 CHT 参数化研究”，不以完整商业 AirJet 反向复刻或双向 FSI 为起点。**

## 1. 项目价值评估

| 维度 | 评级 | 依据 |
|---|---|---|
| 工程价值 | 高 | 薄型电子热管理中，热点温度、空间高度、压损/输入功之间存在真实设计权衡。 |
| 学术可发表性 | 中到高 | 仅做单一稳态云图不够；把脉冲效应、CHT、封装排气和多目标优化合并，并有严格验证，则具备论文主线。 |
| 数据/证据可得性 | 中 | AirJet 公开专利和官方系统资料足够定义“类 AirJet”边界；内部精确尺寸与驱动参数不公开。 |
| 数值可行性 | 高（阶段 A–C） | 稳态 CHT、单 cell URANS 和小阵列均可在普通 Windows 工作站完成。 |
| 数值风险 | 中 | 微尺度转捩、排气边界、TIM 与驱动参数会强烈影响结果。 |
| 全耦合 FSI 可行性 | 低（现在） | 需要膜片材料/尺寸/阻尼/驱动与实验或可靠校准，且计算成本大。 |

### 最重要的判断

项目的科学问题不是“AirJet 到底长什么样”，而是：**在公开可支持的 MEMS 脉冲微射流结构约束下，哪些几何和热设计同时降低热点温度与气动代价？**

这让未知商业细节成为明确的局限性，而非致命缺陷。

## 2. 建议的最小可发表单元（MVP）

### 论文主张

建立并验证参数化 AirJet 类微射流 CHT 模型，量化 `H/D`、孔径/孔距、扩散板与 TIM 对 `Tmax`、`Rth`、`Nu`、`Delta p` 的耦合影响；再以有限数量的周期脉冲案例检验稳态等效模型的偏差。

### 必须具备的证据链

1. 清楚说明公开结构证据和未公开参数的界线；
2. 守恒检查 + 网格独立性 +（若做瞬态）时间步独立性；
3. 至少与一篇一般微射流/合成射流文献的无量纲趋势相符；
4. 不只给最低温度：必须给 `Delta p` 和 `Pfluid` 或输入功代理；
5. 至少一项非平凡比较：稳态等效 vs 脉冲，或均匀阵列 vs 热点定制阵列。

### 不足以发表的版本

- 不验证网格，只展示温度云图；
- 把任意假设频率/振幅称为 AirJet 实际参数；
- 只扫 `H/D`，不报告压损、流量或功率代价；
- 用全阵列双向 FSI 却没有任何参数校准或实验/文献锚点。

## 3. 分阶段计划

| 阶段 | 目标 | 预计有效工作量 | 输出 | 继续条件 |
|---|---|---:|---|---|
| P0 | 资料、变量和 CAD 选择固定 | 1–2 天 | 参数表、几何草图、引用库 | 每个未知量有状态标签 |
| P1 | 单孔稳态 laminar CHT 跑通 | 2–4 天 | 1 个可复跑 case、守恒表 | 能量差 <1%，质量差 <0.5% |
| P2 | 网格与模型敏感性 | 3–5 天 | 三网格表、laminar/SST 比较 | 核心结论不依赖数值误差 |
| P3 | 小型 DOE | 4–7 天 | CSV、响应面、权衡图 | `H/D` 趋势与压损趋势清楚 |
| P4 | 扩散板/TIM/排气优化 | 4–7 天 | Pareto 图、设计建议 | 有可解释的多目标结论 |
| P5 | 单 cell 脉冲瞬态验证 | 5–10 天 | 周期曲线、稳态偏差 | 周期稳定且时间步独立 |
| P6 | 写作与复现包 | 5–8 天 | 论文图表、方法、补充材料 | 从脚本/日志可复跑 |

若 P1/P2 失败，先修正几何、边界或网格；不要跳到 P5。若 P5 时间不足，P0–P4 仍可组成一篇稳态 CHT 优化论文，但必须把“脉冲”留作后续工作。

## 4. 风险登记与处理

| 风险 | 影响 | 早期预警 | 应对 |
|---|---|---|---|
| 产品内部参数不可得 | 高 | 无法为频率/振幅找可靠值 | 用参数化范围，绝不宣称复刻真实产品；做灵敏度分析 |
| 排气边界不真实 | 高 | 温度对出口位置异常敏感 | 从周期域升级为显式排气道，报告边界敏感性 |
| 低 Re/转捩模型不确定 | 中 | laminar 与 SST 结果相差很大 | 报告两模型；检查 Re、近壁网格和文献范围 |
| 动态网格崩溃 | 中 | 负体积、质量泄漏 | 缩小振幅/时间步，先用规定速度边界，采用 layering/overset |
| 计算资源不足 | 中 | 单工况超过 12 小时或内存溢出 | 先单孔/周期扇区；只对 Pareto 点做瞬态 |
| 论文贡献过弱 | 高 | 结果只是常识性单调趋势 | 引入非均匀热点、排气受限或稳态—脉冲差异，保留多目标权衡 |

## 5. Windows 资源与软件决策

Windows 的优势不是系统名称，而是可用的 Fluent/Mechanical 生态、RAM、CPU 核数和许可证。对于 P1–P4，建议至少 32 GB RAM、8 个真实 CPU 核及 SSD；P5 动态网格更适合 64 GB RAM 和更多核。GPU 不是这类 Fluent 微射流/CHT 的首要决定因素；先确认 CPU 求解与内存。

软件优先级：

1. Fluent（主求解器）+ Python 后处理；
2. Mechanical/COMSOL（只在进入膜片模态和单向耦合时加入）；
3. OpenFOAM（若没有 Fluent 许可证，或需要大批量自动化时作为替代）。

## 6. Codex 技能与工具建议

目前官方技能目录没有专门的 CFD/ANSYS 技能，因此不建议为了“有 skill”安装不相关插件。

| 候选 | 建议 | 用途 |
|---|---|---|
| `jupyter-notebook` | 建议安装 | DOE 表、结果 CSV、自动画响应面/Pareto、可复现研究日志 |
| `pdf` | 建议安装 | 最后汇编方法附录、图表报告和可交付 PDF |
| `define-goal` | 可选 | 当项目扩展为跨多周正式目标时，用于锁定里程碑；当前 Markdown 计划已足够 |
| 自定义 `airjet-cfd` skill | P1 完成后再创建 | 固化已验证的 Fluent 设置、网格检查、导出字段和运行日志模板；现在创建会把猜测写死 |

对 Fluent 自动化，优先采用官方 `PyFluent`/Fluent journal，而不是不明来源的“CFD agent skill”。PyFluent 可以从 Python 启动本机 Fluent、调用 TUI/求解器和后处理，但仍要求本机合法 Fluent 安装与许可证。

## 7. 立刻开始时的唯一下一步

在 Windows 打开仓库，读主项目文件和参数 CSV；然后仅完成 P1 的单孔稳态 laminar CHT。首个运行条件固定为：`D=0.30 mm`、`H=1.20 mm`、`Uin=10 m/s`、`Q=5 W`、`Tinf=25 degC`。先导出质量/能量守恒和 `Tmax`；此时不要开始参数扫描、动态网格或 FSI。

当这个算例可复跑后，再让 Codex 创建 Fluent journal、运行日志模板和 Python 后处理脚本。
