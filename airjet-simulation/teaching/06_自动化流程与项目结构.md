# 第6课：自动化流程与项目结构

## 为什么用自动化？

手动操作 ANSYS GUI → 不可重复、容易出错、无法版本控制。

我们的方案：**Python 驱动一切**
- SpaceClaim → Python API
- Fluent Meshing → PyFluent
- Fluent Solver → PyFluent（未来）
- Mechanical → PyMechanical（未来）

## 项目自动化架构

```
airjet-simulation/automation/ansys/
├── approved/                     # 已审核脚本
│   ├── 006/                      # C5
│   │   ├── v03_continuous_fluid_producer.py
│   │   └── v03_pyfluent_watertight_mesh_consumer.py
│   └── 008/                      # P2
│       └── p2_s0_equivalent_plate_producer.py
├── contracts/                    # 合同层
│   ├── v03_finite_throat_contract_v1.py
│   ├── v03_finite_throat_route_v1.json
│   └── test_v03_finite_throat_contract_v1.py
├── run_v03_continuous_fluid_006.py   # Stage 1: CAD
├── run_v03_continuous_mesh_006.py    # Stage 2: 网格
└── test_run_v03_continuous_mesh_006.py
```

## 两阶段流程

### Stage 1: CAD (SpaceClaim)
→ MCP submit_job → SpaceClaim.exe → producer.py → STEP

### Stage 2: 网格 (PyFluent)
→ MCP submit_job → Fluent → consumer.py → .msh.h5

## MCP 工具
- `inventory`, `submit_job`, `poll_job`, `cancel_job`, `artifact_manifest`

## 合同层
每个参数都有合同约束，防止手滑改错。FAIL 闭锁。
